# SPRING_008: Active Orders Query

## Overview

Implement OrderQueryService to efficiently find and retrieve active orders.

## Requirements

1. Retrieve all active orders from the database
2. Support filtering by order status
3. Optimize query performance
4. Use existing repository methods when available

## Constraints

- Use Spring Data JPA repository
- Follow Kotlin coding conventions
- Avoid N+1 query problems
- Minimize database load
